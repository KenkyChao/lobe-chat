# 企业员工使用手册

本文档面向当前企业版分支的实际能力编写，优先说明已经可以从现有界面、配置项和代码行为中确认的功能。仍在调整中的部分不会提前写死，而是统一放在文末的**待确认项**中，便于后续继续补充。

## 适用范围

- **适用对象**：企业版员工用户
- **产品形态**：当前界面品牌、默认入口和基础文案均已切换到 **NaiYun AI**
- **交付边界**：本文只覆盖当前分支中已经明确存在的功能，不替代最终上线公告或售前材料

## 已确认功能

### 1. 品牌与默认入口

- 企业版对外正式名称固定为 **NaiYun AI**
- 默认收件箱入口名称已切换为 **NaiYun AI**
- 应用图标、默认头像与 favicon 已接入企业版品牌资源
- 登录、首页、桌面端欢迎页会沿用同一套品牌名称
- 桌面端与移动端中的 “使用文档” 入口已切换到当前这套企业员工手册

### 2. 账号与登录

- 登录页支持**邮箱或用户名**登录
- 支持**密码登录**、**忘记密码**、**重置密码**、**邮箱验证**
- 当前环境可按实际配置显示 **Magic Link** 与 **SSO / OIDC**
- 登录页会按实际配置显示可用入口，例如 OIDC、飞书、Google、GitHub 等
- 用户可以在个人资料中维护头像、用户名、邮箱，以及已绑定的第三方登录方式
- 登录页底部的提示文案已调整为 “继续即表示你同意遵守本组织的使用规范”
- 登录页底部当前只保留组织使用规范提示，不再额外展示服务协议或隐私政策跳转

<Callout type={'info'}>
  登录方式不是固定写死的。最终看到哪些登录按钮，以你当前账号所在环境实际展示为准。
</Callout>

### 3. 工作区与设置面板

当前企业版已经区分了**个人设置**与**工作区设置**两个层级。

个人设置中，已确认存在以下类别：

- 个人资料、统计、外观、设备
- 快捷键
- 通知（企业特性开启时显示）
- 套餐、用量、积分、账单、推荐奖励（企业特性开启时显示）
- 模型服务商、服务模型、技能、记忆、消息渠道
- 代理、系统工具、存储、进阶配置

工作区设置中，已确认存在以下类别：

- 工作区信息、成员、统计
- 套餐、用量、积分、账单
- 模型服务商、服务模型、技能
- 存储、API Key

另外，当前分支里**消息渠道不在工作区设置中配置**，而是保留在个人设置里统一管理，再按助理选择要绑定到哪个消息频道。

<Callout type={'info'}>
  员工在实际使用时，是否可以看到工作区设置中的全部项目，取决于当前账号被分配到的权限。
</Callout>

### 4. 模型与助理

- 企业版保留了**助理**作为长期工作单元的使用方式
- 用户可以在设置中管理**模型服务商**与**服务模型**
- 正式文档会公开能力范围与使用方式，但**不会写入密钥、访问地址等敏感信息**
- 当前分支已为首页提供一组明确的推荐模型，包含：
  - `Qwen3.6-35B-A3B`
  - `Qwen3.6-27B`
  - `DeepSeek-V4-Flash`
  - `DeepSeek-V4-Pro`
- 当前分支同时保留了 `Qwen3-8B` 作为轻量本地模型选项
- 当前分支中已存在带有 **Private** 标识的私有模型卡片，说明企业版支持接入私有或内网模型能力
- 当前云端模型池已补充图片、视频、语音识别等模型类型，不再只限于聊天模型
- 助理可继续绑定技能、知识、文档与消息渠道

关于模型怎么选、什么场景该用什么模型，请查看专门的模型选择指南。

### 5. 语音输入与语音播报

- 聊天输入框已支持语音转文字
- 当前可选的语音识别通道包括：
  - `Browser`
  - `OpenAI`
  - `OpenRouter`
- 当前默认语音识别模型已切换为 `Qwen3-ASR-1.7B`
- 当前默认语音识别服务已切换为 `OpenRouter`
- 当前默认语音播报模型为 `edge-tts`
- 助理语音播报当前保留单一配置入口，不再展示 Microsoft Speech 等额外服务选项

<Callout type={'info'}>
  如果你只是想把一段语音快速转成文字，优先直接使用聊天输入框的语音按钮即可；如果要让助理输出语音，再到助理设置里调整语音播报配置。
</Callout>

### 6. 消息渠道

- 个人设置中已提供**消息渠道**入口
- 工作区设置中当前不提供独立的消息渠道配置入口
- 正式保留的企业消息平台包括：
  - `QQ`
  - `微信`
  - `飞书`
  - `Lark`
- 渠道接入后的 Bot 绑定发生在**助理 -> 消息频道**中
- 渠道安装结果支持成功提示、失败提示，以及 “已被其他工作区占用” 的拦截说明

<Callout type={'info'}>
  渠道入口和 Bot 绑定是两层配置：先在个人侧完成平台接入，再到具体助理中配置该助理要使用的消息频道。
</Callout>

### 7. 编程助理与桌面能力

- 当前分支已保留并增强了 **Codex** 与 **Claude Code** 这类桌面编程助理能力
- 桌面端可检测本机 CLI，并在缺失时给出安装引导
- 执行设备已包含：
  - **本机**
  - **云端沙箱**
  - **远程设备**
- **云端沙箱**在企业部署中默认可用
- 远程设备通过 `lh connect` 接入
- 系统工具页已存在面向 CLI Agents 的检测与管理入口

## 推荐使用流程

<Steps>
  ### 第一步：确认登录入口

  先观察当前登录页实际提供的方式。若页面显示邮箱、用户名、SSO 或 Magic Link，请直接按组织分配方式登录。

  ### 第二步：进入个人设置检查基础能力

  建议先检查个人资料、设备、模型服务商、服务模型、语音能力和消息渠道，确认当前账号已具备需要的使用权限。

  ### 第三步：检查工作区内可用资源

  如果你的账号能看到工作区设置，可以重点确认当前工作区里已经开放了哪些模型、技能和共享资源。

  ### 第四步：创建或选择助理

  普通业务场景优先从助理开始，按用途区分写作、问答、检索、编程或对外渠道型助理。

  ### 第五步：按场景启用高级能力

  需要桌面协作时使用 Codex 或 Claude Code；需要对外触达时配置消息渠道；需要长期沉淀能力时维护好自己的助理与资料。
</Steps>

## 使用提醒

### 1. 登录入口以页面实际展示为准

- 邮箱密码、邮箱验证、Magic Link、SSO / OIDC 不一定会同时出现
- 如果你在页面上没有看到某种登录方式，说明当前环境未对你的入口开放

### 2. 工作区能力会因账号不同而变化

- 不同员工账号能看到的设置、模型和资源可能不同
- 如果某个功能在文档中提到，但你当前页面没有显示，优先以实际界面为准

### 3. 渠道配置带有个人归属

- 消息渠道需要先由你自己完成接入
- 接入完成后，再到具体助理中决定是否启用该渠道

### 4. 语音入口与模型会因环境配置变化

- 语音识别服务可以切换为 Browser、OpenAI 或 OpenRouter
- 默认值已切换到 `OpenRouter + Qwen3-ASR-1.7B`
- 如果你当前页面没有看到某个语音入口或模型，优先以实际界面为准

## 限制说明

- **桌面能力不等于 Web 通用能力**：Codex、Claude Code、本机执行与系统工具依赖桌面端
- **远程设备依赖接入状态**：若未在目标机器执行 `lh connect`，则不会显示可用远程设备
- **登录入口与渠道列表不是固定常量**：请以当前页面实际展示为准
